# Contributing to Cybersecurity Enthusiasts from 42

Thank you for your interest in contributing to this organization.
We are a community of 42 Network students and alumni focused on **cybersecurity, CTF challenges, and open-source security projects**.
Your contributions help the community grow and share knowledge.

---

## Ways to Contribute

- **CTF Writeups** — Share solutions, methodology, and lessons learned.
- **Guides and Notes** — Upload study materials, roadmaps, and useful references.
- **Security Tools** — Develop or improve open-source tools for research and training.
- **Bug Reports** — Report issues in tools, scripts, or documentation.
- **Feature Suggestions** — Propose new features or improvements.

---

## How to Contribute

1. **Fork** the repository you want to contribute to.
2. **Create a new branch**:
   ```bash
   git checkout -b feature/your-feature
   ```
3. **Make your changes** (writeups, code, or documentation).
4. **Commit** with a clear, descriptive message:
   ```bash
   git commit -m "Add: CTF writeup for PicoCTF challenge"
   ```
5. **Push** your branch:
   ```bash
   git push origin feature/your-feature
   ```
6. **Open a Pull Request** with a clear description of your contribution.

---

## Guidelines

- Use clear, plain English so that all skill levels can follow along.
- For writeups: include the challenge name, category, and a step-by-step solution.
- For code: add comments, a README, and usage examples where applicable.
- Be respectful — this is a learning community open to all experience levels.

---

## Important

- Share **educational content only**. Malicious or illegal material is strictly prohibited.
- Respect the **42 Network code of conduct** and community guidelines.
- All contributions must adhere to **ethical hacking principles**.
